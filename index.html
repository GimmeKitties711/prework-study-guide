HTML

  • HTML is the code behind the functionality and formatting of a website.
  • The goal is to have the website be laid out in a pleasant way that is beneficial to the user.
  • The head element contains information about the webpage.
  • This information is mostly invisible to the user, but it still gives important directions to the browser.
  • The body element represents the visible content shown to the user.
  • The body can include special content like pictures and links to other websites.

CSS

  • CSS allows us to decide how we want to style our website; we can decide on colors, font sizes, and page layouts.
  • Attributes that have the same settings can be decided by the same set of brackets.
  • This efficient way of coding is called Do Not Repeat Yourself (DRY) programming.
  • The most efficient method is external CSS, where the CSS file is separate from the HTML file.
  • The CSS file needs to be linked to the HTML file for the visual changes to take effect.
  • A class can be created in the CSS file, and all elements of that class in the HTML file will be affected by the class's properties.
  • A margin indicates how much space we want around the outside of an element.
  • A padding indicates how much space we want around the content inside an element.

Git

  • The purpose of Git is to establish a connection between the local repository on your computer and the remote repository on GitHub.
  • To transfer local changes to the server, a pull request is made, and then it is reviewed and approved by colleagues before the changes are merged with the main branch.
  • cd: change directory
  • pwd: print working directory (to make sure you are in the right place in your files)
  • ls: lists all of the files and folders in a directory
  • git status: checks what branch we are currently on
  • git checkout -b branch-name: creates a new branch and switches to it
  • Similarly, git checkout branch-name switches to an existing branch without creating a new one.

JavaScript

  • JavaScript is the part of web development that allows a user to interact with a website.
  • The console can be used to display important messages about the website.
  • A variable is a named container that allows us to store data in our code.
  • A for loop can be used as an efficient way to iterate through a repetitive action without having to type a unique line for each instance.
  • A function can be used to perform a certain set of actions whenever we call its name.
  • Simply defining a function is not enough to make it run; it has to be called.
  • Control flow is the order in which a computer executes code in a script.
  • JavaScript is read from top to bottom, so you have to define variables before writing out actions pertaining to them.
